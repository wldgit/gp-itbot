# Резервное копирование данных

## 1. Назначение документа

Документ определяет правила резервного копирования (backup) корпоративных данных для предотвращения их потери и обеспечения возможности восстановления.

## 2. Область применения

Распространяется на все информационные системы, сервисы и данные компании.

## 3. Основные принципы

- Резервное копирование должно выполняться регулярно
- Данные должны быть доступны для восстановления
- Копии должны храниться отдельно от основных систем
- Должна обеспечиваться защита резервных копий

## 4. Типы резервного копирования

Используются следующие типы:

- Полное (full backup)
- Инкрементальное (incremental backup)
- Дифференциальное (при необходимости)

## 5. Частота резервного копирования

Рекомендуется:

- Критичные системы — ежедневно
- Важные данные — несколько раз в неделю
- Остальные данные — по необходимости

Конкретное расписание определяется IT-отделом.

## 6. Хранение резервных копий

- Копии должны храниться:
  - в отдельной инфраструктуре
  - в защищённом хранилище
- Рекомендуется использовать правило 3-2-1:
  - 3 копии данных
  - 2 разных носителя
  - 1 копия вне основной инфраструктуры

## 7. Безопасность резервных копий

- Резервные копии должны быть защищены от несанкционированного доступа
- При необходимости используется шифрование
- Доступ к backup ограничивается

## 8. Проверка восстановления

- Необходимо регулярно проверять возможность восстановления данных
- Проводить тестовое восстановление
- Фиксировать результаты проверок

## 9. Ответственность сотрудников

Сотрудники обязаны:

- сохранять рабочие данные в утверждённых системах
- не хранить важные данные локально без необходимости
- сообщать о потере данных

## 10. Запрещённые действия

Запрещается:

- хранить единственную копию данных локально
- отключать механизмы резервного копирования
- самостоятельно изменять настройки backup без согласования
- хранить резервные копии в незащищённых местах

## 11. Инциденты

При потере данных необходимо:

- немедленно сообщить в IT-отдел
- указать, какие данные были утеряны
- не предпринимать самостоятельные попытки восстановления (если не согласовано)

## 12. Ответственность

Несоблюдение политики резервного копирования может привести к потере данных и нарушению работы компании.
