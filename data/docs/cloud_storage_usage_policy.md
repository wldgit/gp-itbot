# Работа с облачными хранилищами

## 1. Назначение документа

Документ определяет правила использования облачных хранилищ для хранения, передачи и совместной работы с корпоративными данными.

## 2. Область применения

Документ обязателен для всех сотрудников, использующих облачные сервисы в рабочих целях.

## 3. Разрешённые сервисы

Для работы допускается использование только утверждённых сервисов, например:

- Google Drive
- Microsoft OneDrive

Использование личных облачных хранилищ запрещено.

## 4. Общие правила

- Хранить рабочие файлы только в корпоративных хранилищах
- Использовать структуру папок, принятую в компании
- Не создавать дубли файлов без необходимости
- Использовать понятные имена файлов

## 5. Управление доступами

- Доступ к файлам предоставляется по принципу минимальной необходимости
- Использовать групповые доступы вместо индивидуальных, где возможно
- Регулярно проверять выданные доступы
- Удалять доступ при увольнении или смене роли сотрудника

Запрещается:

- открывать доступ «для всех по ссылке», если это не требуется
- выдавать доступ внешним пользователям без согласования

## 6. Работа с файлами

Рекомендуется:

- использовать совместное редактирование вместо пересылки файлов
- хранить актуальную версию файла в одном месте
- избегать копий вида: final_v2_final3.docx

## 7. Безопасность данных

- Не загружать в облако конфиденциальные данные без необходимости
- Использовать ограничение доступа для чувствительных файлов
- Проверять права доступа перед отправкой ссылок

## 8. Передача файлов

- Передавать файлы через корпоративное облако, а не через вложения в почте
- Проверять получателя перед отправкой
- Ограничивать срок действия доступа при необходимости

## 9. Запрещённые действия

Запрещается:

- использовать личные облачные сервисы для рабочих задач
- хранить пароли и чувствительные данные в открытом виде
- предоставлять публичный доступ без необходимости
- скачивать и распространять данные без разрешения

## 10. Аудит и контроль

Рекомендуется:

- периодически проверять структуру хранилища
- удалять устаревшие файлы
- контролировать права доступа

## 11. Ответственность

Нарушение правил может привести к утечке данных и дисциплинарным мерам.
