# Основы кибербезопасности для сотрудников

## 1. Назначение документа

Документ описывает базовые правила кибербезопасности, которые должны соблюдать сотрудники при работе с корпоративными системами, устройствами и данными.

## 2. Область применения

Правила распространяются на всех сотрудников, подрядчиков и пользователей, имеющих доступ к корпоративной инфраструктуре.

## 3. Основные принципы

- Использовать только разрешённые корпоративные сервисы
- Не передавать учётные данные другим пользователям
- Проверять подозрительные письма, ссылки и вложения
- Сообщать об инцидентах в IT-отдел
- Соблюдать принцип минимально необходимого доступа

## 4. Пароли и доступы

- Пароли должны соответствовать требованиям документа `password_policy.md`
- Минимальная длина пароля — 8 символов
- Для критичных сервисов рекомендуется использовать более длинные пароли
- По возможности должна быть включена двухфакторная аутентификация

## 5. Фишинг и подозрительные сообщения

Признаки подозрительного письма:

- срочная просьба выполнить действие
- неизвестный или странный отправитель
- вложение, которое не ожидалось
- ссылка на внешний сайт
- просьба ввести пароль или код подтверждения

При получении подозрительного сообщения сотрудник не должен переходить по ссылкам или открывать вложения. Сообщение необходимо передать в IT-отдел.

## 6. Безопасность рабочих устройств

Сотрудник обязан:

- блокировать компьютер при уходе с рабочего места
- не устанавливать неразрешённое ПО
- не отключать антивирус, firewall и другие средства защиты
- своевременно устанавливать обновления
- не подключать неизвестные USB-устройства

## 7. Работа с корпоративными данными

- Хранить рабочие данные только в утверждённых местах
- Не копировать корпоративные данные на личные устройства
- Не передавать конфиденциальную информацию через личную почту или мессенджеры
- Проверять получателей перед отправкой файлов и сообщений

## 8. Работа в сети

- Не использовать публичные Wi-Fi сети для работы без дополнительной защиты
- Для удалённого доступа использовать только утверждённые способы подключения
- Не подключаться к неизвестным сетям без необходимости

## 9. Инциденты безопасности

Нужно немедленно сообщить в IT-отдел, если:

- потеряно устройство
- замечен несанкционированный доступ
- пароль мог быть скомпрометирован
- получено подозрительное письмо
- система ведёт себя необычно

## 10. Запрещённые действия

Запрещается:

- передавать пароли другим пользователям
- использовать один аккаунт несколькими сотрудниками
- устанавливать пиратское или неразрешённое ПО
- отключать средства защиты
- хранить рабочие данные в личных облачных сервисах

## 11. Ответственность

Несоблюдение правил кибербезопасности может привести к утечке данных, нарушению работы систем и дисциплинарным мерам.
