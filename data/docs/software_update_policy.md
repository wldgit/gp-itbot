# Обновление и поддержка программного обеспечения

## 1. Назначение документа

Документ определяет правила обновления и поддержки операционных систем и программного обеспечения для обеспечения безопасности и стабильной работы.

## 2. Область применения

Распространяется на все рабочие устройства и серверы компании.

## 3. Основные принципы

- ПО должно быть актуальным
- Обновления должны устанавливаться своевременно
- Уязвимости должны устраняться как можно быстрее
- Используется только разрешённое ПО

## 4. Типы обновлений

- Обновления безопасности (критичные)
- Функциональные обновления
- Исправления ошибок (patches)

## 5. Порядок обновления

- Обновления устанавливаются централизованно или по регламенту
- Критичные обновления устанавливаются в приоритетном порядке
- Перед установкой обновлений может выполняться тестирование

## 6. Обязанности сотрудников

Сотрудник обязан:

- не отключать обновления
- перезагружать устройство при необходимости
- не откладывать установку обновлений без причины
- сообщать о проблемах после обновлений

## 7. Ограничения

Запрещается:

- самостоятельно устанавливать неразрешённое ПО
- отключать службы обновлений
- использовать устаревшие версии программ без согласования

## 8. Серверные системы

- Обновления выполняются по согласованному расписанию
- Перед обновлением создаётся резервная копия
- Обновления тестируются при возможности

## 9. Контроль и мониторинг

- IT-отдел контролирует актуальность ПО
- Используются средства централизованного управления
- Проводятся регулярные проверки

## 10. Инциденты

Необходимо сообщить в IT-отдел, если:

- обновление вызвало сбой
- программа перестала работать
- система работает нестабильно после обновления

## 11. Ответственность

Несоблюдение политики обновлений может привести к уязвимостям, сбоям и нарушению безопасности.
